#  requirements.txt  —  Render Production (API server only)
#
#  This file is INTENTIONALLY minimal. It contains only what the
#  live query API needs. Ingestion, evaluation, and data-prep
#  dependencies are in requirements-dev.txt and are run locally.
#
#  Pinning strategy:
#    - Top-level packages: pinned to exact version (==)
#    - Conflict zone (httpx, httpcore, click, anyio): pinned explicitly
#      to prevent pip backtracking. These four packages are pulled
#      transitively by chromadb, cohere, groq, and fastapi with
#      overlapping but non-identical ranges. Explicit pins let pip
#      skip the resolver loop entirely.
#
#  Build command for Render:
#    pip install --no-cache-dir --prefer-binary -r requirements.txt
# ════════════════════════════════════════════════════════════════
